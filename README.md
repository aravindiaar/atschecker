# ResumePro — AI-Powered Resume Builder & ATS Checker

ResumePro is a full-stack web application that helps job seekers create professional resumes and optimise them for Applicant Tracking Systems (ATS). It combines a live resume editor with an AI-driven analysis and rewriting engine.

## Features

### Resume Builder
- Build a resume from scratch using a structured two-panel editor
- Fill in your name, title, location, contact details, professional summary, work experience, skills, and education
- Live preview updates in real time as you type
- Choose from three templates: **Professional**, **Technical**, and **Modern**
- Download the finished resume as a **DOCX** file

### ATS Checker & Resume Fixer
A guided 4-step workflow:

1. **Upload your resume** — supports PDF, DOCX, and TXT files (up to 10 MB)
2. **Analyse** — paste an optional job description for a targeted keyword match, or run a general ATS scan without one
3. **Fix with AI** — GPT-powered rewrite of your summary, skills, and experience bullets to improve ATS compatibility (never fabricates experience you don't have)
4. **Re-analyse** — see your before/after scores side by side

#### Scoring Breakdown (with a Job Description)
| Component | Weight | What it measures |
|---|---|---|
| Keyword Match | 50% | How many JD keywords appear in your resume |
| Format & Structure | 25% | Contact info, standard section headings, bullet points |
| Experience | 25% | Years of experience and seniority level vs. JD requirements |

#### Scoring Breakdown (general scan, no JD)
| Component | Weight |
|---|---|
| Keyword Match | 40% |
| Format & Structure | 35% |
| Experience | 25% |

## Tech Stack

### Frontend (`artifacts/resume-builder`)
- **React 19** + **Vite**
- **Tailwind CSS v4**
- **shadcn/ui** component library
- **TanStack Query** (React Query) for data fetching — hooks auto-generated from the OpenAPI spec via Orval
- **docx** — client-side DOCX generation for resume downloads

### Backend (`artifacts/api-server`)
- **Express 5** on **Node.js 24**
- **OpenAI API** (GPT-4) for AI resume improvements
- **pdf-parse** — PDF text extraction
- **mammoth** — DOCX text extraction
- **multer** — file upload handling
- **Zod** — request/response schema validation
- **Drizzle ORM** + **PostgreSQL** — database layer

### Shared Libraries (`lib/`)
| Package | Purpose |
|---|---|
| `api-spec` | OpenAPI 3.1 specification (source of truth) |
| `api-zod` | Zod schemas generated from the OpenAPI spec |
| `api-client-react` | TanStack Query hooks generated by Orval |
| `db` | Drizzle ORM schema and database client |
| `integrations-openai-ai-server` | Pre-configured OpenAI SDK wrapper |

## API Endpoints

### `POST /api/resume/parse`
Upload a resume file (PDF, DOCX, or TXT) and receive extracted plain text.

**Request:** `multipart/form-data` with a `file` field
**Response:** `{ text: string, filename: string }`

---

### `POST /api/resume/ats-check`
Analyse a resume against an optional job description and return an ATS score with detailed feedback.

**Request:**
```json
{
  "resumeText": "...",
  "jobDescription": "..." // optional
}
```

**Response:**
```json
{
  "overallScore": 72,
  "keywordScore": 60,
  "formatScore": 90,
  "experienceScore": 70,
  "matchedKeywords": ["python", "aws", "ci/cd"],
  "missingKeywords": ["java", "mysql"],
  "strengths": ["Strong keyword alignment..."],
  "suggestions": ["Add a dedicated Technical Skills section..."]
}
```

---

### `POST /api/resume/fix`
Use AI to rewrite and improve a resume based on ATS analysis results.

**Request:**
```json
{
  "resumeText": "...",
  "jobDescription": "...",       // optional
  "missingKeywords": ["java"],   // optional
  "suggestions": ["..."]         // optional
}
```

**Response:**
```json
{
  "improvedSummary": "...",
  "improvedSkills": "...",
  "experienceImprovements": [
    { "index": 0, "improvedBullets": ["..."] }
  ],
  "overallChanges": "...",
  "improvedResumeText": "..."
}
```

## Getting Started

This project uses **pnpm workspaces**. Make sure you have `pnpm` installed.

```bash
# Install dependencies
pnpm install

# Start the API server
pnpm --filter @workspace/api-server dev

# Start the frontend
pnpm --filter @workspace/resume-builder dev
```

Set the following environment variables before starting the API server:

| Variable | Description |
|---|---|
| `OPENAI_API_KEY` | Your OpenAI API key (required for AI fix feature) |
| `DATABASE_URL` | PostgreSQL connection string |
| `PORT` | Port for the API server (default: 8080) |

---

## Notes

- The AI fix will never fabricate skills or experience. It only improves the wording and structure of what is already in your resume.
- For very mismatched resumes (low keyword overlap with the JD), the score improvement after an AI fix will be naturally limited, since keyword matching accounts for 50% of the JD-mode score.
